# Title Message
# <type>(<scope>): <subject>
# ex) solved(BOJ1001): A+B 문제 풀이
# -> 50자 이내, 마침표 없이 작성 (명령문 형태 권장)

# 한 줄 공백. subject와 body 구분

# Body Message (선택사항)
# - 무슨 문제를 어떤 방식으로 접근했는지 간략히 설명
# - 로직 선택 이유, 고려한 반례 등도 포함 가능
# - 72자 기준 줄바꿈

# Footer (선택사항)
# 이슈 관리가 필요한 경우만 작성
# ex) Resolves: #123

# --- COMMIT END ---

# ✅ Type 종류 (Title의 <type>)
#   feat    : 새로운 문제 풀이 추가
#   fix     : 기존 풀이 수정 (오답 수정, 로직 개선 등)
#   docs    : README, 주석, 문제 설명 추가/수정
#   chore   : 파일 정리, 디렉토리 구조 변경 등
#   style   : 코드 스타일 수정 (공백, 들여쓰기 등)

# ✅ Scope 예시 (괄호 안 <scope>)
#   BOJ1000, PGMS_문자열압축, SWEA_1234 등 문제 번호/이름

# ✅ Footer 종류
#   Resolves: #이슈번호      → 해당 이슈 자동 종료
#   Ref: #이슈번호           → 참고용 이슈
#   Related to: #이슈번호    → 관련 이슈 참조

# ✨ 커밋 메시지 팁
# - subject는 무엇을 했는지를 "명확히" 작성
# - 영어일 경우 첫 단어는 대문자, 마침표 ❌
# - body는 왜 했는지/무엇을 고려했는지를 중심으로 작성
# - 필요 시 코드 구조 변경 이유 등 설명
